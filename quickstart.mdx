ExcaliDash runs as two Docker containers — a backend API server and a frontend Nginx server — coordinated with Docker Compose. The fastest path to a running instance is the Docker Hub deployment, which uses pre-built images and requires no local build step.

<Warning>
  ExcaliDash is in BETA. Use TLS, a trusted reverse proxy, fixed secrets, and regular backups before exposing it to production traffic.
</Warning>

## Prerequisites

- **Docker** (with the Docker Compose plugin v2 — `docker compose` not `docker-compose`)
- A machine with an open port `6767` (or a reverse proxy in front)

## Deploy with Docker Hub

<Steps>
  <Step title="Download the Compose file">
    Pull the production Compose file directly from the repository:

    ```bash
    curl -OL https://raw.githubusercontent.com/ZimengXiong/ExcaliDash/main/docker-compose.prod.yml
    ```
  </Step>

  <Step title="Pull the images">
    Download the latest backend and frontend images from Docker Hub:

    ```bash
    docker compose -f docker-compose.prod.yml pull
    ```
  </Step>

  <Step title="Start ExcaliDash">
    Start both containers in detached mode:

    ```bash
    docker compose -f docker-compose.prod.yml up -d
    ```

    The backend starts on internal port `8000`. The frontend is accessible at `http://localhost:6767`.
  </Step>

  <Step title="Open the dashboard">
    Navigate to [http://localhost:6767](http://localhost:6767) in your browser. You'll see the ExcaliDash dashboard and can immediately create your first drawing.

    If you have authentication enabled, check the backend logs for the one-time bootstrap setup code:

    ```bash
    docker compose -f docker-compose.prod.yml logs backend --tail=200 | grep "BOOTSTRAP SETUP"
    ```
  </Step>
</Steps>

## What's running

| Container | Image | Port | Role |
|-----------|-------|------|------|
| `excalidash-frontend` | `zimengxiong/excalidash-frontend:latest` | `6767` (public) | Serves the React UI; proxies API and WebSocket to backend |
| `excalidash-backend` | `zimengxiong/excalidash-backend:latest` | `8000` (internal) | REST API, SQLite database, real-time collaboration |

The backend data (SQLite database and generated secrets) is stored in the `backend-data` Docker volume, which persists across container restarts and upgrades.

## Build from source (alternative)

If you want to run from a local clone instead of Docker Hub images:

```bash
git clone https://github.com/ZimengXiong/ExcaliDash.git
cd ExcaliDash

docker compose build
docker compose up -d
```

This uses `docker-compose.yml` (not `docker-compose.prod.yml`) and builds images locally.
